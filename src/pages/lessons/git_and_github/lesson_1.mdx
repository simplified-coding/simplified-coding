<h2 style="text-align: center;"> Εισαγωγή στο Git & GitHub </h2>

---
<h2 style="text-align: center;"> <span style={{ color: 'yellow', fontWeight: 'bold' }}> Τι είναι το Git και γιατί είναι σημαντικό; </span> </h2>

Το Git είναι ένα σύστημα διαχείρισης εκδόσεων (Version Control System - VCS) 
που χρησιμοποιείται ευρέως από προγραμματιστές για την παρακολούθηση και τον έλεγχο της ανάπτυξης λογισμικού.
Είναι σχεδιασμένο να αποθηκεύει, να διαχειρίζεται και να καταγράφει τις αλλαγές στον κώδικα πηγαίου κώδικα κατά 
τη διάρκεια της ανάπτυξης λογισμικού.

Κάποια από τα βασικά χαρακτηριστικά του Git περιλαμβάνουν:

1. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Ιστορικό Αλλαγών: </span> Το Git διατηρεί αναλυτικό ιστορικό όλων των αλλαγών που έχουν γίνει στον κώδικα.

2. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Διακλαδώσεις (Branches): </span> Οι προγραμματιστές μπορούν να δημιουργήσουν διακλαδώσεις από τον κώδικα πηγαίου κώδικα,
επιτρέποντάς τους να εργαστούν παράλληλα σε διάφορες λειτουργίες χωρίς να επηρεάζουν τον κώδικα του κύριου έργου.

3. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Συγχώνευση (Merge): </span> Το Git επιτρέπει τη συγχώνευση των διακλαδώσεων, δηλαδή την ενσωμάτωση των αλλαγών από μια διακλάδωση πίσω στον κύριο κώδικα.

4. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Απομακρυσμένα Αποθετήρια (Remote Repositories): </span> Το Git υποστηρίζει τη συνεργασία μεταξύ ανθρώπων που βρίσκονται σε διάφορες τοποθεσίες.
Ο κώδικας αποθηκεύεται σε απομακρυσμένα αποθετήρια, όπου μπορεί να χρησιμοποιηθεί από πολλούς προγραμματιστές.


Η σημασία του Git αναδεικνύεται από τα εξής:

1. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Εύκολη Συνεργασία: </span> Επιτρέπει σε προγραμματιστές από διάφορες ομάδες και τοποθεσίες να συνεργαστούν αποτελεσματικά στον κώδικα χωρίς σύγχυση.

2. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Παρακολούθηση Αλλαγών: </span> Το Git επιτρέπει την ακριβή παρακολούθηση όλων των αλλαγών που έχουν γίνει στον κώδικα, κάτι που βοηθάει στην εύρεση σφαλμάτων και τη διαχείριση του έργου.

3. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Ασφάλεια και Ακεραιότητα: </span> Ο κώδικας αποθηκεύεται ασφαλώς και μπορεί να ανακτηθεί σε περίπτωση απώλειας δεδομένων ή σφάλματος.

4. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Ελευθερία για Δοκιμές: </span> Οι προγραμματιστές μπορούν να δημιουργούν προσωρινές αλλαγές και να τις δοκιμάζουν χωρίς να επηρεάζουν τον κύριο κώδικα.

Συνολικά, το Git αποτελεί ένα απαραίτητο εργαλείο για τον ανάπτυξη λογισμικού και τη διαχείριση του κώδικα,
επιτρέποντας αποτελεσματική συνεργασία και παρακολούθηση των αλλαγών.

---
<h2 style="text-align: center;"> <span style={{ color: 'yellow', fontWeight: 'bold' }}> Τι είναι το GitHub και πως συγκρίνεται με το Git; </span> </h2>

Το GitHub είναι μια πλατφόρμα διαχείρισης κώδικα που χρησιμοποιείται για τη συνεργασία σε έργα λογισμικού που χρησιμοποιούν το Git.
Το GitHub έχει πολλές λειτουργίες και εργαλεία που βοηθούν στη διαχείριση του κώδικα και την επικοινωνία μεταξύ των προγραμματιστών μαζί με το Git.
Μερικά απο αυτά είναι:

1. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Διαχείριση Κώδικα: </span> Το Git είναι το σύστημα διαχείρισης εκδόσεων που χρησιμοποιείται για την παρακολούθηση και τον έλεγχο του κώδικα πηγαίου κώδικα.
Το GitHub παρέχει μια πλατφόρμα όπου οι προγραμματιστές και η ομάδα ανάπτυξης μπορούν να αποθηκεύσουν, να διαχειριστούν και να συνεργαστούν σε έργα Git.

2. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Αποθετήρια (Repositories): </span> Στο GitHub, ο κώδικας αποθηκεύεται σε αποθετήρια, τα οποία είναι χώροι όπου οι προγραμματιστές αποθηκεύουν τον κώδικά τους.
Τα αποθετήρια GitHub χρησιμοποιούν το Git για να παρακολουθούν και να ελέγχουν τις αλλαγές στον κώδικα.

3. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Συνεργασία: </span> Ένα από τα βασικά πλεονεκτήματα του GitHub είναι η δυνατότητα συνεργασίας.
Προγραμματιστές από όλον τον κόσμο μπορούν να συνεισφέρουν στον κώδικα πηγαίου κώδικα, να προτείνουν αλλαγές και να σχολιάζουν τον κώδικα.

4. <span style={{ color: '#ADD8E6', fontWeight: 'bold' }}> Παρακολούθηση Έργων: </span> Το GitHub παρέχει εργαλεία για την παρακολούθηση της εξέλιξης των έργων,
τη διαχείριση των εκδόσεων και τη δημιουργία ζητήσεων ενσωμάτωσης (Pull Requests) για την ενσωμάτωση νέων αλλαγών.

Συνολικά, το GitHub διευκολύνει τη συνεργασία, την κοινοποίηση και τη διαχείριση του κώδικα πηγαίου κώδικα σε ομάδες ανάπτυξης και κοινότητες προγραμματιστών.

<h3 style="text-align: left;x"> Εγκατάσταση Git </h3>

<p style="text-align: left;"> [Windows](https://git-scm.com/download/win)  [Mac OS](https://git-scm.com/download/mac) </p>

#### Linux
Για την εγκατάσταση του Git σε υπολογιστές με Linux, τρέξτε τη σωστή εντολή για τη διανομή σας:

* Debian/Ubuntu: `apt-get install git`
* Fedora: `yum install git (up to Fedora 21)`, `dnf install git (Fedora 22 and later)`
* Gentoo: `emerge --ask --verbose dev-vcs/git`
* Arch Linux: `pacman -S git`
* openSUSE: `zypper install git`
* Mageia: `urpmi git`
* Nix/NixOS: `nix-env -i git`
* FreeBSD: `pkg install git`
* Solaris 9/10/11 (OpenCSW): `pkgutil -i git`
* Solaris 11 Express: `pkg install developer/versioning/git`
* OpenBSD: `pkg_add git`
* Alpine: `apk add git`

<h2 style="text-align: center;"> <span style={{ color: 'yellow', fontWeight: 'bold' }}> Εγγραφή στο GitHub </span> </h2>

Για την εγγραφή σας στο GitHub

1. Μπείτε στην [ιστοσελίδα του GitHub](https://github.com/)
2. Πατήστε το κουμπί **Sign up**
3. Εισάγετε το email σας
4. Εισάγετε έναν κωδικό πρόσβασης
5. Δημιουργήστε ένα όνομα χρήστη
6. Επιλέξτε `y` αν θέλετε να λαμβάνετε ενημερωτικά email από το GitHub και `n` αν δε θέλετε
7. Εισάγετε τον κωδικό μιας χρήσης που είναι στο email σας
8. Πατήστε continue και continue for free
9. Ολοκληρώσατε την εγγραφή σας!
